几个术语

变量

变量来源于数学，是计算机语言中能储存计算结果或能表示值抽象概念。变量可以通过变量名访问。在指令式语言中，变量通常是可变的；但在纯函数式语言（如Haskell）中，变量可能是不可变（immutable）的。——百度百科

上面的定义比较生涩拗口，我觉得 方骥老师 的比喻更加生动贴切。

这里的变量就像方程式里的 x, y，一开始你列了方程，但是暂时还没法算出一个数，之后只要把x变成具体的数字就可以算出一个值了。

过程

网上关于过程的定义：过程组被作为一个整体，指示Excel中如何执行特定任务执行的语句。执行的任务可以非常简单或非常复杂的，这是一个很好的做法，把程序分成较小的部分。

模块

网上对模块的定义：模块是其中代码被写入的区域。

VBA编辑窗口中的左侧有一个工程栏，里面显示的都是工程项目，在列表中是一个个模块。

变量

为什么要使用变量

运用变量来编写带有变动参数的语句代码，在为这些变量赋值之后就可以具体执行这些语句。

常用变量类型

  • Variant（万用型）
  • Date（日期）
  • Boolean（布尔类型/逻辑值）
  • Object（对象）
  • String（字符串）
  • Integer（整数）
  • Long（长整数）
  • Single（单精度浮点数）

    Excel中的Integer的范围是从负的30,000+到正的30,000+，并不包含所有的整数。因此如果要涉及到绝对值更大的数值，就要使用Long类型

不同级别的变量

过程

最常见的在子程序的编写过程中申明的变量就是过程级别的变量。过程级别的变量只有在当前自己编写的这个Sub中会使用，而不会和其他的Sub共享。

模块

在模块中申明的变量可以在不同的过程之间传递。简单的来说就是，在一个编辑窗口的最顶部（所有的Sub之上）申明的变量就是模块级别的变量。

公共

模块列表中有不同的文件夹，其中在Excel对象文件夹下的模块和在模块文件夹下的模块是有区别的，后者是全局类型的模块，在后者编辑窗口中可以申明公共变量。公共变量是全局变量，可以在不同的模块之间传递。

过程

ExcelVBA的过程有三种：

  1. 函数Function
  2. 子程序Sub
  3. 属性Property

这三种过程里最后一种不太常见，前面两种有一个明显的不同之处：

  • Function函数除了执行特定的语句外还会返回一个特定的结果
  • Sub则只是执行语句，不会返回一个结果

ExcelVBA中对象的属性和方法

发表于 2016-07-10   |   分类于 闻叨

什么是属性和方法

上面一篇文章中讲到了对象，简单的概括就是我们在Excel中对什么东西进行操作，这个什么东西就是所谓的对象。
现在讲到的属性和方法我们也可以打个比方。假如我们现在就谈论水果，那我们知道人会有品种；价格；口感等等标签来帮助我们区分不同的水果，这就是属性。而如果我们想要对它进行一些动作，比如洗水果;切开水果；吃水果等等的时候，这些词就是我们讲到的方法。
概括的来说：

  • 属性就是一个对象的一些标签
  • 方法就是我们要对对象进行的动作（或操作）

如何知道一个对象有哪些属性和方法

从上面的文章我们可以知道Excel中有很多的对象，这些对象的属性和方法又不近相同，怎么样才能知道一个对象有哪些属性和方法呢？难道我们要上网去找相应的表格，然后一个个背下来或者对着去找吗？其实巨硬公司早就想到了这一点，当我们进入到VBA编辑界面的时候，我们可以试着在一个对象的.后面单击右键，选择“属性/方法列表”，之后就可以看到一条下拉菜单，里面就是当前这个对象的属性和方法了。其中图标像一块被丢出去的绿色砖头的就是方法，而图标像一只手指着一个列表的就是属性，一动一静，还算好理解。

需要注意的是，当我们的对象是Selection的时候是无法出现方法/属性列表的，因为Selection是泛指被选择的对象，VBA还没有智能到能够自己判断之前到底选择了什么，所以无法给出对应的列表。

Excel VBA 对象

发表于 2016-07-10   |   分类于 闻叨

对象

是什么

所谓对象就是我们要对什么东西进行操作。例如，我们要对单元格进行填充上色，又例如我们复制当前一个工作表等等。
之前完成的VBA里我们仅仅在VBA的编辑窗口里改动了第一行和最后一行，给整个代码块套上了一个循环结构，但是中间的代码究竟代表什么我们没有去深究。
当我们理解了对象之后，我们就可以读懂中间的代码究竟对哪些东西进行了操作和修改。

具体分类


主要有以下几种类型：

  • Application
  • Workbooks
  • Sheets
  • Worksheets
  • Charts
  • Range

Application

Application指的是整个运行程序，当我们打开一个或者几个Excel表格文件的时候，这里的application指的就是Excel。

Workbooks

WorkBooks指的是我们在Excel表格中创建的工作表的合集。

Sheets

Sheets这个单词可能比起前几个会看起来比较眼熟，因为我们每次新建一个Excel的时候，左下角的页面标签就是Sheets1。Sheets就是工作表，与前面的工作簿有一字之差。就跟日常生活中一样，“表”是一张一张的，当很多张“表”被钉在一块，就成了一本“簿”，我们也可以这样来理解 Sheets 和 WorkBooks 的区别。

Worksheets

这里又出现了一个Worksheets，字面意思是工作表。那是不是我们选中的表就叫工作表呢？并不是的，其实Excel中有很多种的表格类型，当我们右键单击标签，选择插入时，会出先很多种表格类型让我们选择。

在图中我们可以看到除了“工作表”以外，还有“图表”和“宏表”等很多类型的表。我们平常用的最多的就是“工作表”，这里的Worksheets也特指“工作表”这一类型的表格。而上面提到的Sheets则泛指所有类型的表。

Charts

原理与上面的Worksheets差不多，Charts就是特指“图表”类型的表。

Range

Range就是单元格的意思，比起上面的几个词来说应该是比较好理解的了。

《你的知识需要管理》读书笔记

发表于 2016-07-04   |   分类于 闻叨

个人知识管理

  • 学习知识
  • 保存知识
  • 共享知识
  • 使用知识
  • 知识创新

    1. 学习知识

    学什么？

  • 知识工作者要有根基，一些通识知识是与人沟通，掌握信息，学习其他知识的基础。
  • 在确定自己学习方向时需要做减法：即聚焦。想做到聚焦，还要先了解自己，然后细分自己的兴趣领域，努力在某一个细分的领域做到专家。
  • 为什么要做到专家？现在知识更新周期很短，若是仅仅知道某个领域的入门知识，过不了多长时间，你掌握的这些知识就会变成大家的通识，很容易失去竞争力。
  • 知识的架构：

    这其中基础知识作为学习的根基，是最需要得到重视的，所以在入门阶段应当重视学习的方法和资料的来源，不要让自己基础不牢。

    怎样学习

  • 先要知道，学习最大的成本——Time
  • 由此我们引出学习的决策模型
  • 一步步入门
  1. 打好基础：阅读经典教材，论文和该领域的一些文章
  2. 了解行业：查看搜索引擎的前50页有助于对行业有快速整体的了解
  3. 跟进发展：阅读相应期刊，行业的论文，可能的话多和专家交流
  4. 实践创新：在工作中利用自己掌握的知识，尽力把知识显性化

2. 保存知识

  • 先摒弃陋习：家藏n G书。没有节制的保存会让知识毫无用处，因为没有经过大脑的知识是无法发挥出来的。
  • 说完无节制的保存，再说组词语的区别：数据、信息和知识
  1. 数据：单纯的数据毫无意义，就是几个数字
  2. 信息：信息富裕数据环境，让数据变得有意义
  3. 知识：可以用来决策和行动的信息
  • 要保存哪些符合目前和将来发展方向的知识（如读此书的时候作者推荐的很多工具都已经过时，我便将理论知识保留下来，工具的推荐不做保留），并建立索引，确保日后“书到用时可拿到”
  • 要建立索引，就要有良好的保存习惯（文件夹的深度不宜超过三层，文件要有系统的命名）
  • 文件夹的命名：“标题-机构（或作者）-分类-类型”

    e.g. ppt模板-德勤-个人工具-ppt

共享知识

为什么？

  • 让别人知道你有才华
  • 然别人学到东西，别人更容易认可你

好处

  • 共享出来是大脑对知识的再整理，这样才能真正掌握知识
  • 结实高质量的朋友

    看一个人的身价，看他的对手；
    看一个人的底牌，看他的朋友。

范围

  • 同事、朋友 => 专业、行业人士 => 社会范围
  • 要记着你想把知识共享给谁，这些人最习惯于哪些方式
  • 在单位内做知识共享是知识共享的基础，单位内才能给予你更多重视和尊重，不要仅仅热衷于在互联网上包装自己，这样是本末倒置的。

工具

  1. 单位IT系统
  2. 出版书籍
  3. Blog
  4. 维基（Wiki）
  5. 微博

使用知识

为什么？

  • 仅有显性知识是不够的，还要了解社会和具体工作，隐形知识也起到关键作用

    显性知识：能够用语言、文字、肢体等方式表达清楚的知识
    隐性知识：虽然知道如何做，却很难告诉别人

  • 隐性知识的特点：

  1. 对自己隐，但是对他人而言是显性知识的话，我们应当反思自己的学习方法是否走了弯路，可以在他人的基础上学习
  2. 隐性知识并非永远是隐形的
  3. 谁最先做到显性化，谁就是新的知识战场的开拓者，会占有先机
  4. 隐性知识显性化是一种能力差别
  5. 外部环境对显性化有影响
  • 有时无知也是有好处的，它能给我们尝试的勇气
  • 团队协作才能使知识的价值最大化

使用知识的方式

  1. 找份工作
  2. 申请专利
  3. 将知识具体化为方案或者服务或者机器

提升个人价值的方法

  • 影响因素
  1. 知识的独特性
  2. 社会对知识的需求

总结

  • 学习知识要求精，要重根源
  • 保存知识则要断舍离，要有章法
  • 分享是为了让别人认可自己，这是双赢
  • 使用知识是为了让自己掌握，所有的纸上谈兵都是空

资产的基本概念

资产定义

资产是能够用货币表示的经济资源，能够用货币表示的经济资源就属于资产。

  • 资产定义的限制
  1. 有的无形资产会被忽略在报表之外
  2. 人力资源的价值衡量是个难题，因为：
    一、企业招到人才≠买到人才
    二、服役时间难以确定

资产的分类

流动 vs. 非流动

  • 区分的概念：能否在一年之内变现
  • Special rule：有的行业生产周期相对较长，不能简单按照变现周期划分流动资产与非流动资产（e.g. 飞机制造业；船舶制造业；房地产；etc. 这些企业的存货虽然变现周期长，但也是流动资产）

流动资产

包括如下：

  1. 货币资金
  2. 债券
  3. 存货

非流动资产

包括如下：

  1. 长期投资类
  2. 固定资产类
  3. 无形资产类

经营 vs. 投资

  • 区分概念：对利润的贡献方式（钱是对内使用还是对外使用）

经营资产

  • 与企业的日常经营活动有关
  • 包括：
  1. 货币资金
  2. 债权
  3. 存货
  4. 固定资产
  5. 无形资产

投资资产

  • 对外投资形成的资产
  • 包括：
  1. 交易性金融资产
  2. 可供出售金融资产
  3. 持有至到期投资
  4. 长期股权投资

资产分类的总结

把资产分为 投资&经营 比 流动&非流动 对人们理解报表的帮助更大。我认为这是因为企业本身的使命就是盈利，前者围绕企业对企业盈利的贡献来进行分类，显然更符合分析的逻辑。相较之下，后者仅仅靠资产表面的性质来划分，并无太大的帮助。
