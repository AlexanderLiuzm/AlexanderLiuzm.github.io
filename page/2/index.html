提取汉字，英文或数字

发表于 2016-12-29   |   分类于 闻叨

为什么要提取字符

数据分析中的工作流程分为很多个阶段，其中的有一个阶段就是提取数据，而各种各样的字符串，数字，汉字，英文都是数据的一种。

提取

目标

我们的目标是分别把C列中的数字、汉字和英文，并分别将其存放在每行后面的空单元格中。

思路

在一个单元格当中有很长的字符串，一个接一个把字符提取出来，并且进行检验，是否为中文，是否为英文，是否为数字，然后把符合要求的数据提取出来，并存放到空单元格当中。

实战

创立循环

这个比较简单，已经无数次使用双层循环进行遍历操作。外层循环用来遍历每一行的单元格，内层循环用来遍历单个单元格中字符串的每个字符。

判断

判断这一步要分很多情况讨论，我们要分别讨论字符是否是数字、汉字或英文。

汉字

首先想要把单个字符取出进行判断，我们要用到Mid()函数，之后d要判断一个字符是否是中文，我们要用到Asc()函数。中文的Asc()函数值小于0，我们可以利用这一点来进行判断。

数字

要判断一个字符是否是数字，我们可以使用一个更方便的方法——Like。这个方法是用来判断字符的相似性，如果相似就得到TRUE，相反的，和后面的字符不相似就得到FALSE。我们要判断一个字符是否是数字，可以判断它是否和0~9当中的某个数字相似，可以写成[0-9]。

赋值

批量处理合并单元格

发表于 2016-12-23   |   分类于 闻叨

为什么要处理合并单元格

因为在数据处理当中一维的表格处理起来最为方便，可以汇总为二维的表格或者其他自己想要的表格。但矛盾就在这里，合并单元格大大降低了表格的可处理性，无论是筛选，排序还是统计（sumif），都会产生错误，对后续的处理操作带来极大的麻烦。因此在数据处理操作之前要对合并单元格进行处理。

目标

让合并的单元格都取消合并，并且都填充对应的值。

思路

循环法

先把合并单元格进行恢复，取消合并。再对一列单元格进行逐一的检验。如果单元格内含有数据，那就不做处理；如果单元格内没有数据，那就把这个单元格的上一列的数据赋值给它。（举例来说，令B5.Value = B6.Value）但是为了不让这个循环一直进行下去，要给循环设定一个界限，就是常用的Usedrange.rows.count。

定位法

定位法相比循环法更为简单，先取消合并（此时会选中原先合并单元格的区域），然后定位空单元格，将这些单元格的值都等于他们的上一行的值就可以了。

不同表头的多表合并

发表于 2016-10-09   |   分类于 闻叨

目标

这次VBA的目标就是把不同工作表内的不同表头（即列标题有所不同）的表格进行汇总到一张表内。

先来看下这张汇总表格有什么地方是值得注意的：

  1. 列标题与数据是一一对应的，来自哪个表的数据就对应行标题来自哪个表格
  2. 列标题汇总了所有的列标题（项目名称），避免了重复
  3. 数据填充在相应的单元格，没有数据的地方就留空

思路

因为列标题是汇总的，没有重复的，所以就先想到可以利用字典来进行汇总，同时排除了重复项。又因为数据要填充到相应的单元格，即要有对应的行号和列号，就选择用列标题的值和列标题对应的item进行标号，以确保数据能进入正确的单元格。利用一个循环找到操作一张表格的起始行号和结束行号，对第一列进行填充，以达到粘贴的数据与行标题是对应的效果。

简而言之，分为以下几步：

  1. 遍历工作表，取出不同的标题行名称
  2. 遍历每一列，把每一列的数据复制到合并表
  3. 命名对应的行标题

实战

遍历工作表

建立一个字典，对一个文档中的所有工作表进行遍历，对凡是名字不叫合并的表，取出首行的标题名称，检查是否在字典关键字中，若没有就添加到关键字，并指定item。item的命名是有讲究的，因为可以直接用来对应粘贴数据值。由于数据是从第二列开始粘贴的（第一列是显示对应的工作表名称），所以item的命名也从2开始。最后把字典中的关键字（不同的标题名称）赋值给合并表中的首行单元格。

复制数据

同样使用循环遍历所有的工作表，只是将中间的字典部分进行了更换。把一张工作表中的数据都粘贴到合并表中，其中粘贴的内容要向下横移一行（offset），因为首行是标题行，不需要粘贴。

合并表中的标题行是唯一值，所以序列号与单张表中的标题行序列肯定是不匹配的，如何确保特定列的内容能准确无误的粘贴进相应的列当中呢？这就要用到我们之前为关键字指定的item，通过keys查找item的值，我们可以确定对应的列在合并表中是哪一列，以进行定位。

确定粘贴的行数就相对简单些，因为只要在使用过的行以下进行粘贴就好了。但是要注意的是由于我们是对一个文档中所有的工作表进行遍历循环，所以行数的增加（r = newst.UsedRange.Rows.Count + 1）应当放在遍历列循环的外面，否则每完成一列的粘贴，行数就会往下错，导致数据粘贴成阶梯形状（自己动手试试或者脑补吧 =_=）

命名行标题


最后利用r和r2来确定遍历一个工作表的起始行和结束行，将这几行的第一列命名为相应的表名即可。

可以看出二维表与一维表有很大的不同，二维表中每个数据都对应着两个字段名称，而一维表中值对应自己这一行的数据。简单说，看二维表总要纵看一次横看一次，而一维表只用横着看一次就行。

在《三张表格走天下》中我读到过小蚊子老师这样的总结，做数据分析需要三张表，一张是参数表，一张是录入表，最后一张是透视表。其中包括最多的原始数据的录入表就要采用一维表格的形式。为什么呢？因为一维表格更加的简单，更加适合采用筛选、排序等方法对数据进行提取和加工。

一维数据表在数据分析的工作中有重要的作用，但是普通人似乎更习惯二维表，因为生活中见得最多的还是二维表。如何进行转换呢？我们的思路是：

这里所说的数据的类型并不是单元格格式，单元格格式有很多类型，比如日期类型，文本类型和自定义等等。但是数据的类型是指输入的值自己的性质，不是单元格的格式，最常用到的有数据类型和文本类型。

除了可以利用Excel中的删除重复项的函数改造得到的宏进行删除重复项，还可以通过在VBA构建一个‘字典’来进行删除重复项。

如何使用字典删除重复项

两种方法的区别

两种方法最大的差别就是是否直接对单元格进行删除操作。
字典并不会直接对单元格进行删除操作，我们仅仅是利用字典只能保存唯一值的特点进行中间过渡，把所有唯一值都保存到字典数组中去，然后再把字典中的唯一值输出赋值给单元格。我们进行的操作仅仅是写入和赋值。
Excel删除重复项函数则是一次次循环对比，对满足条件的行进行删除行的处理，而不仅仅是对字符串的读写了。

相对来说，字典比Excel删除重复项操作要更快，更有效率一些。这是为什么呢？主要是因为对字符串的处理只是文字级别上的，而对单元格的删除处理已经达到窗口级别。（好吧，我承认这是我编的，我自己这么理解，反正对单元格的操作层级更高，需要更多的计算和处理）计算机对操作字符串会更在行，更有效率。
